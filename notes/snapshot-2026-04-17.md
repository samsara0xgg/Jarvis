## 1. 架构总览

|------|------|------|
| `jarvis.py` | 主入口（唤醒/Enter 两种模式） | 71KB 单文件 |

| `data/` | 本地模型（sensevoice / silero_vad / speechbrain cache） | ~1.1GB 总 |
| `ui/` | OLED 显示（RPi ST7789 / Mac 模拟） | 可选 |

### 主启动流程（`jarvis.py`）

```
run_always_listening()
  └── OpenWakeWord "hey_jarvis" 持续监听
      └── 触发 → handle_utterance()
          ├── [并行] SpeakerVerifier（声纹 ECAPA-TDNN）
          ├── [并行] SpeechRecognizer（SenseVoice INT8 → Whisper fallback）
          └── _process_turn()  ← 语音/文本两条路共享
              ├── 告别词 (120ms 短路)
              ├── 记忆快捷键 / 升级词 / 学习检测
              ├── 关键词自动化规则
              ├── DirectAnswer（memory cosine > 阈值 → 跳 LLM）
              ├── [并行] IntentRouter (Groq LRU-256) + MemoryQuery
              ├── LocalExecutor（本地指令）or LLMClient（云端流式 tool-use）
              ├── TTS pipeline（MiniMax → edge-tts → pyttsx3）
              └── 后台：记忆抽取 + 行为日志 + Trace
```

关键文件：`jarvis.py:handle_utterance` / `jarvis.py:_process_turn` / `core/interrupt_monitor.py`（全双工打断）。

## 3. config.yaml 关键开关

> 共 500+ 参数，按"能不能跑"分级。

### ⚠️ 影响"能否启动"（必配）

| Section | Key | 说明 |
|---------|-----|------|
| `asr` | `provider: sensevoice` + `sensevoice_model_dir: data/sensevoice-small-int8` | 无模型文件 → ASR 崩 |
| `audio` | `vad_model_path: data/silero_vad.onnx` | 无文件 → VAD 崩 |
| `speaker` | `model_source: speechbrain/spkrec-ecapa-voxceleb` | 首次跑会下载 ~800MB |
| `llm.presets` | `fast` / `deep` 的 API key env var | 无 key → LLM 不可用 |
| `tts.engine` | `minimax` （默认） | 需 `MINIMAX_API_KEY`，否则走 fallback |
| `devices.mode` | `sim` 或 `live` | live 需 Hue 配对 |

### ⚙️ 影响"体验"（可微调）

- `audio.vad_db_threshold: -45.0` — dBFS 门控（注意：曾被误用 SPL 单位，**已修正**）
- `audio.vad_prob_threshold: 0.4` / `vad_required_hits: 3` / `vad_required_misses: 24`
- `verification.threshold: 0.70` — 声纹严格度
- `tts.stream_player.enabled: true` — 零间隙流播 vs subprocess afplay
- `wake_word.enabled: true/false` — 真唤醒 vs `--no-wake` Enter 模式
- `interrupt.enabled` — 全双工开关
- `scheduler.enabled` — APScheduler 定时

### 硬件假设

- **采样率 16kHz** 硬核绑定：ASR / 声纹 / VAD 模型全 16k，**不能改**
- **麦克风/扬声器**：未硬编码 index，用 sounddevice 默认（macOS 需授权）
- **Hue Bridge**：假设局域网 HTTPS + 自签证书
- **GPU**：`speaker.device: cpu` + ONNX CPU，无 CUDA 自动检测
- **OLED**：RPi 时 I2C/SPI；Mac 时 `emulator: true`

## 5. Daily-Use 缺口（要每天实际用的角度）

### 🔴 阻塞启动的前置

| 项 | 检查方法 | 恢复 |
|----|---------|------|
| `GROQ_API_KEY` env | `echo $GROQ_API_KEY` | 无 → 意图路由崩 |
| `XAI_API_KEY` 或 `OPENAI_API_KEY` env | 同上 | 无 → LLM 崩 |
| `MINIMAX_API_KEY` env | 同上 | 无 → TTS 走 edge-tts 降级 |
| `data/sensevoice-small-int8/` | `ls data/sensevoice-small-int8` | 无 → ASR 崩 |
| `data/silero_vad.onnx` | `ls data/silero_vad.onnx` | 无 → VAD 崩 |
| `.cache/speechbrain/` | 首次跑自动下载 ~800MB | 需联网 + 耐心 |
| macOS 麦克风权限 | 系统设置 → 隐私 → 麦克风 | 终端/Python 需勾选 |

### 🟡 首次运行会卡住的点

| 症状 | 原因 | 解法 |
|------|------|------|
| 首句 ASR 延迟 2s | SenseVoice 冷启动 | 已内置 `_prewarm_connections` + executor warmup |
| Groq timeout | API 限额/限流 | 用 `--no-wake` 直接测 LLM |
| MiniMax 30s 等待 | TTS 限流 | fallback 链能接住，但影响体验 |
| Hue live 模式失败 | Bridge 未配对 | 改 `devices.mode: sim` 或跑 `setup_hue.py` |
| 录音立刻返回空 | macOS 麦克风权限未授权 | 系统设置开权限 |

### 🟢 无硬编码陷阱（代码自查）

- 无硬编码 IP / 路径 / 麦克风 index（全从 config.yaml 读）
- 所有外部依赖都有 fallback（TTS 5 引擎 / LLM Cerebras / ASR Whisper）
- Hue 有自动发现 + HTTP 回退
- `permission_manager` 统一拦截设备操作（符合 CLAUDE.md 规则）

### ⚠️ Daily-Use 已知缺失（从 code 看）

| 项 | 现状 |
|----|------|
| systemd 服务文件 | 无，需手工 `python jarvis.py &` |
| 自动重启 | `core/health.py` 只探测不拉起 |
| 日志轮转 | `logs/` 存在但无 rotation，长期跑会撑爆磁盘 |
| config 热重载 | 无，改 config.yaml 必须重启 |
| 电源管理 | 无（RPi 上 always-listening 吃 CPU） |

### 📋 两种运行模式（CLAUDE.md）

```bash
python jarvis.py --no-wake    # dev，按 Enter 录音
python jarvis.py              # prod，"Hey Jarvis" 唤醒
python -m pytest tests/ -q    # ~1059 tests
python system_tests/runner.py --mode cc --suite general    # E2E 烟测
```

## 6. 一句话总结

Jarvis 的**框架完整度远超过实际使用成熟度**：
- 代码侧：架构清晰、功能广、测试齐全、无硬编码陷阱
- 运行侧：缺的是**环境就绪度**（API key / 模型文件 / 音频权限）+ **长期运行设施**（日志轮转 / systemd / 热重载）

下一步做的不是写新功能，而是**把这几道启动前置问题一次扫干净**，然后 `python jarvis.py --no-wake` 真的跑起来，在用的过程中调参 debug。
